Functionalitati

Websiteul creat de noi este un instrument de testare si de stresare pentru aplicatii Web.

Utilizatorul va avea posibilitatea detectarii:
-mărimii inadecvate a unor fişiere
-aspectelor legate de performanta
-aspectelor legate de calitatea codului
-headere
-statistici disponibile şi prin intermediul unui serviciu Web

Mai mult, se vor putea alege diverse scenarii de vizitare automată a sitului pentru a se studia punctele mai dificil de accesat.

Diagrama arhitecturala

User scenario

Orice utilizator care isi creeaza cont poate folosi aplicatia.

  • Utilizatorul introduce linkul websitului despre care vrea sa afle date.
  • Datele vor fi distribuite pe categorii, astfel utilizatorul poate alege sa se afiseze doar anumite informatii.
  • Se vor oferi resurse de informare pentru testele care nu au un rezultat tocmai pozitiv.

Module functionale

  • Sign up.
  • Log in.
  • Afisare calitate cod, headere, compresii, legaturi invalide.
  • Afisare aspecte de performanta preluate cu Apache Benchmarks.

Used technologies

  • Am folosit Figma pentru user flow, UI/UX
  • Aplicatia este creata folosind PHP
  • Serverul folosit este Apache
  • Baze de date SQL pentru conturile utilizatorilor.
  • Partea de client va include HTML, CSS si JavaScript

API

  • API mysql pentru conectarea la baza de date.
  • CSS Validator
  • Markup Validator
  • https://api.hackertarget.com/pagelinks - pentru afisarea tutoror linkurilor accesibile dntr-o pagina
  • https://api.hackertarget.com/httpheaders - pentru afisarea headerelor dintr-o pagina
  • HTML APIS (<input>,<nav>)
  • Apache Benchmarks

Views

  • O fereastra pop-up care contine textul “Passwords don't match!” in cazul in care la sign up fieldurile de password si repeat passworn nu corespund.

Etapele intermediare ale dezvoltarii

  • Proiectarea si interfata pentru afisarea rezultatelor proiectate.
  • Legatura cu baza de date pentru sign up si log in.
